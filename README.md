# Event-Application-SNHU-CS-360

Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?

The app I developed was designed to help users organize and manage events in a simple, accessible way directly on their phone. The main goal was to provide a clean and functional interface where users could view, add, and manage event-related information without unnecessary complexity. This app addressed the user need for a straightforward, local solution to keep track of events without relying on external services.

What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?

To support these needs, the app included key screens such as a main event list screen, input screens for adding or editing events, and structured UI components like card views to display information clearly. The UI design focused on readability, spacing, and intuitive navigation so users could quickly understand how to interact with the app. Keeping users in mind meant making sure elements were not cluttered, buttons were easy to find, and data was presented in a logical way

How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?

When coding the app, I approached development incrementally by building and testing one feature at a time instead of trying to implement everything at once. I focused on structuring the layout first, then connecting data to the UI, and finally refining functionality.

How did you test to ensure your code was functional? Why is this process important, and what did it reveal?

To ensure the app was functional, I tested features frequently using the emulator and debugging tools. I verified that user inputs were handled correctly, screens displayed the right data, and interactions worked as expected. This process is important because it helps catch issues early and ensures the app behaves reliably. Testing also revealed small UI issues and data flow problems that I might not have noticed otherwise, allowing me to fix them before finalizing the app.

Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?

Throughout the development process, I had to innovate when working with UI layout and component sizing, especially when adjusting card views to display information properly across the screen. Finding the right balance between layout structure and visual design required experimenting with different settings and approaches. This challenge pushed me to better understand how UI components behave and how small changes can significantly impact the overall design.

In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience?

One component where I was particularly successful was designing and implementing the event display using card views. This part of the app demonstrated my ability to combine layout design, data presentation, and user experience principles. It showed that I could not only make features work but also present them in a way that is visually clear and user-friendly, reflecting both my technical skills and my growth as a developer.
